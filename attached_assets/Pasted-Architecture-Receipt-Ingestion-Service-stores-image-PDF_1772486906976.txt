Pipeline

Upload receipt -> create receipts row (INGESTED).
OCR run -> populate receipt_extractions, set OCR_DONE.
Exact matcher:
Same currency
Amount exact (or within 0.01)
Date in [receipt_date-1, receipt_date+1]
Vendor normalized exact or alias exact
Exact outcomes:
1 hit -> create link (EXACT), set AUTO_LINKED.
0 or >1 hits -> run candidate scorer.
Candidate scorer:
Search transactions in date window (default ±7 days; configurable)
Exclude transactions already linked (unless allowed)
Compute weighted score and store top N in receipt_match_candidates.
Threshold decision:
score >= 0.90 and (top1-top2) >= 0.10 -> auto-link (SCORED_AUTO)
Else -> NEEDS_REVIEW.
Review queue resolution:
Manual link suggested/other txn
Create new transaction from OCR
Mark ignore/duplicate
Scoring formula (example)

score_total = 0.55*amount + 0.25*date + 0.20*vendor
amount score
exact: 1.0
within 0.01: 0.95
within 1%: 0.75
else decays to 0
date score
same day: 1.0
±1 day: 0.8
±3 days: 0.5
±7 days: 0.2
vendor score
canonical exact: 1.0
alias exact: 0.9
fuzzy ratio normalized (rapidfuzz): 0..1
API contracts

POST /receipts
input: file
output: {receipt_id, status}
POST /receipts/{id}/process
triggers OCR+matching
output: {status, auto_linked, transaction_id?}
GET /receipts/review?status=NEEDS_REVIEW
output: list of receipts with extraction summary
GET /receipts/{id}/candidates
output: ranked candidates with score breakdown
POST /receipts/{id}/link
input: {transaction_id}
output: link result
POST /receipts/{id}/create-transaction
input: optional overrides
output: {transaction_id, link_id}
POST /receipts/{id}/ignore
input: {reason}
output: status update

Rules/guards

No auto-link when:
OCR amount/date confidence below threshold
Multiple near-identical candidates
Transaction already linked and duplicates disallowed
Idempotency key for processing endpoint to avoid duplicate links.
Full audit log for every decision path.

Tuning plan

Start conservative thresholds.
Track manual actions 2–4 weeks.
Adjust weights/thresholds from acceptance data.
Expand merchant aliases from accepted manual links.
Acceptance criteria

Exact unique matches auto-link correctly.
No-match receipts always appear in review queue.
Review shows top 3–5 plausible transactions with transparent reasons.
Users can create+link new transaction in one flow.
All outcomes are auditable and reversible.
